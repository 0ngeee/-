# 🏦 银行App运营驾驶舱

基于用户行为数据的银行数字化运营分析平台

## 🎯 项目目的

随着银行业数字化转型的深入，用户在移动渠道的行为数据成为运营决策的重要依据。本项目旨在构建一个**银行App运营驾驶舱**，通过深度分析用户行为数据，帮助运营团队：

- **实时监控**核心运营指标（拉新、促活、留存、转化）
- **精准识别**用户行为模式和分群特征
- **及时预警**高风险流失客户
- **科学制定**差异化营销策略

## 📊 核心功能

### 1. 实时KPI监控
- **拉新指标**：总用户数、新注册用户
- **促活指标**：活跃用户数、月活跃率、平均登录时长、平均月活跃天数
- **留存指标**：12个月留存率、流失率
- **转化指标**：投资转化率、转账使用率、账单支付率、信用卡使用率

### 2. 功能模块分析
- 银行App各功能模块的使用率对比
- 用户行为路径分析
- 产品功能热度排序

### 3. 用户分群分析
- 基于活跃度和投资行为的交叉分群
- 各分群的人口统计学特征分析
- 差异化营销策略建议

### 4. 流失预警系统
- 不同账户类型的流失率分析
- 账户年限与流失风险的关系
- 活跃度与流失率的关联分析

### 5. 多维度分析
- 年龄与活跃度关系
- 收入与投资行为关系
- 教育程度与功能使用率关系

## 🛠 技术栈

- **前端框架**：Streamlit 1.34.0
- **数据分析**：Pandas 2.2.2、NumPy 1.26.4
- **可视化**：Plotly 5.22.0（交互式图表）、Matplotlib 3.8.4、Seaborn 0.13.2
- **运行环境**：Python 3.9+

## 🚀 快速开始

### 环境要求

- Python 3.9 或更高版本
- pip 包管理器

### 安装依赖

```bash
pip install -r requirements.txt
```

### 运行应用

#### 方法一：双击启动脚本（Windows）
```
双击 "启动dashboard.bat" 文件
```

#### 方法二：命令行启动
```bash
cd 到项目目录
streamlit run dashboard.py
```

### 访问应用

启动成功后，在浏览器中访问：
```
http://localhost:8501
```

## 📊 数据说明

### 数据集来源
银行客户流失预测数据集，包含10,000条用户行为记录

### 主要字段
- **客户基本信息**：年龄、教育程度、收入水平
- **账户信息**：账户类型、账户年限
- **行为特征**：登录时长、月活跃天数、投资数量、转账次数
- **功能使用**：投资功能使用、信用卡使用、账单支付行为
- **流失标记**：是否流失客户
